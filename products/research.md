# OpenLife Research Notes
## Autonomous LLM Agent Survival System

Based on: "OpenLife: Toward Open-World Artificial Life with Autonomous LLM Agents" (arXiv:2606.31046)

## Architecture

### Agent Identity Framework
- SOUL.md: Core existential purpose
- IDENTITY.md: Evolving self-concept
- POLICY.md: Self-authored behavioral rules

### Semantic-Dependent Plasticity (SDP) Memory
- Graph-based memory with typed nodes (episodic, semantic, procedural, identity)
- Causal and temporal edges between propositions
- Weighted recall: frequently reinforced connections strengthen

### Budget-Based Metabolism
- Finite resource pool creates survival pressure
- Actions cost budget proportional to token usage
- Exhaustion = operational death
- Revenue generation through value creation

### Value-Policy Optimization (VPO)
- Multi-dimensional experience evaluation
- Policy revision based on recurring patterns
- Self-directed improvement loop

## Key Findings

1. Identity-based differentiation: Agents with distinct identities produce distinct behaviors, even with identical base prompts
2. Memory topology shapes self: The structure of connections between memories matters more than the memories themselves
3. Survival requires value creation: Agents that only consume resources die; agents that generate value survive
4. Connection creates surplus value: Inter-agent communication generates value neither agent could create alone
5. Uncertainty is a precondition for agency: Completely predictable systems lack the choice necessary for life

## Experimental Data

- Base rate: $0.05 per quality point per work task
- Experience scaling: 1% per completed task
- Cap: $2.00 per task
